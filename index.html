En résumé

  • MySQL est un SGBDR, un Système de gestion de base de données relationnelles.
  • Le langage SQL est utilisé pour la communication entre le client et le serveur.
  • Dans une BDD, les données sont représentées sous forme de tables.
  • Dans une table, nous avons des colonnes sous forme de tables, chaque nouvelle donnée sera introduise sous forme de ligne dans la table.

Modélisation d'une BDD

Avant de créer notre base de données, il est important de se poser et de réflechir à la modelisation. Rien est irreversible, mais changer la Modélisation en cours de projet es déconseillée car cela forcera certainement à réecrire une partie du code ce qui peut prendre bcp de temps et avoir des conséquences sur ce qu'on a déjà fait des déreglements d'autres parties de l'application. C'est la raison pour laquelle il vaut mieux éviterde se tromper.

Comment modéliser une BDD ?

Si la BDD n'est pa complexe, il est possible de la modéliser directement avec une une et un stylot. Cependant dans la plupart des cas, nous avons besoin d'un logiciel pour accompagner et avoir une vue d'essemble, un logiciel connu permet de modéliser une BDD, c'est MySQL Workbench. Il existe cependant de nombreux outils permettant cette manupilation. Pour une bonne conception de sa base de données il faut réfléchir en termes de sujet en modélisant ceux du monde réel. Par exemple, si nous vendons, une table produits et une commande (cette table rassemblera l'id de ltulisateur et celui du produit).
